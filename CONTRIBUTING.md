# Contributing to hcloud-k8s

Thank you for your interest in contributing!

## Guidelines for PRs

1.  Fork the repository.
2.  Create a new branch for your feature or bugfix.
3.  Ensure all tests pass (`make test`).
4.  Run the linter (`make lint`).
5.  Submit a pull request with a descriptive title and body.

## Issues

Please report bugs or feature requests using GitHub Issues. Provide as much detail as possible.

## Coding Standards

-   Follow Go best practices (Effective Go).
-   Use `gofmt` to format your code.
-   Ensure all exported functions and types have comments.
-   Write tests for new functionality.
